汇编语言基础

汇编语言指令（就列几个常见的）

mov（MOVe） & lea（Load Effective Address）

mov a,b表示将b寄存器的值赋给a，相当于C语言中的a=b，在实际理解的时候，如果寄存器是（栈）指针，可以理解为将指针指针a移动到指针b的位置

lea a,b用于加载一个操作数的内存地址（b）到一个寄存器（a）中。

; 辨析：
mov rax, [addr] ; 将内存中地址为addr的值赋给rax
lea rax, [addr] ; 将addr赋给rax

add & sub & sar（Shift Arithmetic Right） & sal（Shift Arithmetic Left） & shr（Shift RIght） & shl（Shift Left）

add和sub显然，不说

sar 的作用是将一个二进制数向右移动指定的位数，并在最左侧（最高位）补齐原始最左侧的位，即符号位。 这个操作在处理有符号数时非常重要，因为它能够保持数值的符号不变，从而正确地处理正负数的位移操作。 算术右移一位相当于将数值除以2，同时保持符号位不变。

sal 指令将一个二进制数向左移动指定的位数，并在最右侧（最低位）补齐零。对于有符号数来说，这相当于将数值乘以2的相应次幂，同时保持符号位不变。

shr 指令用于将寄存器或内存中的值向右移动指定的位数。与算术右移（sar）不同，逻辑右移在移位时会在左侧（最高位）补零，而不是补符号位。

shl 指令用于将寄存器或内存中的值向左移动指定的位数。在逻辑左移操作中，被移出的位（从最低位）会被丢弃，而左侧（最高位）空出的位会被填充为0。

push（PUSH） & pop（POP）

这两个是最基础的栈操作，push a表示将a寄存器的值压入栈中，pop a表示从栈中弹出一个值，并将其存到a寄存器中

jmp（JuMP）

jmp的作用是进行无条件跳转。它使程序的执行流程跳转到指定的内存地址，从那里继续执行指令。jmp 指令可以用来实现程序中的分支和循环控制。

leave（LEAVE）

相当于mov sp,bp+pop bp

ret（RETurn）

相当于pop ip。在计算机体系结构中，IP寄存器（Instruction Pointer）是一个特殊的寄存器，它的主要作用是存储下一条将要执行的指令的地址。

jz（Jump if Zero）

它主要用于判断某个寄存器或内存中的值是否为零，如果为零则跳转到指定的代码位置继续执行，否则继续顺序执行下一条指令。

二进制文件

可执行文件的定义

广义：文件中的数据是可执行代码的文件：.out、.exe、.sh、.py

狭义：文件中的数据是机器码的文件：.out、.exe、.dll、.so

可执行文件的分类

Windows：PE（Portable Executable） Linux：ELF（Executable and Linkable Format）
可执行程序 .exe .out
动态链接库 .dll .so
静态链接库 .lib .a

ELF文件格式

ELF文件的类型包括：可执行文件（.exec）、可重定位文件（.rel）、共享目标文件（.dyn）

在审视一个目标文件时，有两种视角可供选择：

链接视角：通过节（Section）来划分，用来划分功能（In Disk）

运行视角：通过段（Segment ）来划分，用来划分读写执行权限（In Memory）

由Linker完成节到段的转化

下面是一些重要的节：

  1. .got（Global Offset Table）：
    • .got 是全局偏移表，用于动态链接。它包含指向动态分配内存地址的指针。当程序需要访问全局变量或静态变量时，通过 .got 表来获取正确的地址。
  2. .plt（Procedure Linkage Table）：
    • .plt 是过程链接表，用于处理动态函数调用。它包含一系列跳转指令，用于首次调用外部函数时解析函数地址，并将解析后的地址存储在 .got 表中。
  3. .symbol（符号表，通常是 .symtab 或 .dynsym）：
    • .symbol 表保存程序中的符号（如函数名、变量名）及其对应的地址。.symtab 用于调试和符号解析，包含所有符号；.dynsym 用于动态链接，包含需要动态解析的符号。

栈迁移

栈迁移（Stack Pivoting）是一种在漏洞利用中常见的技术，用于改变栈的正常行为，特别是在缓冲区溢出等安全漏洞的情况下。栈迁移技术可以用于解决栈溢出空间大小不足的问题。通过将栈指针劫持到攻击者控制的内存处，攻击者可以在相应的位置进行 ROP 操作。

综合来说，栈迁移能够实现的效果有：

配合输入函数使用，实现任意地址写

溢出空间不足时，变相增加溢出长度

栈迁移的核心在于修改栈指针（如 x86 架构中的 esp 和 ebp），使其指向攻击者控制的内存地址。这样，攻击者可以在新的栈空间中执行 ROP 链或其他攻击代码。

栈迁移主要利用 leave; ret 这两个指令。leave 指令的作用是恢复栈帧，即将 ebp 的值赋给 esp，然后 pop ebp。ret 指令则是将栈顶的值弹入 eip，即下一条指令的地址。

在函数执行完毕后，通常会执行 leave 和 ret 指令来恢复到函数执行之前的状态。栈迁移的利用中，需要使用两次 leave 指令来达到将 ebp 转移到我们构造的后门函数所在的起始地址。

首先通过栈溢出覆盖 prev ebp 的值，让程序在执行完当前函数后执行 leave（mov esp, ebp; pop ebp）恢复栈帧时，获取到错误的 prev ebp 从而让 ebp 跳转到指定的位置。

在实际操作中，攻击者需要先在数据段或 bss 段构造 gadget 代码段，然后通过栈溢出构造垃圾数据和 gadget 代码段所在的地址以及程序中的 leave_ret 指令地址。执行 leave 和 ret 指令后，ebp 和 esp 指针被劫持，按照正常的函数调用栈的过程执行攻击者的 gadget 代码片段。

需要注意的一些小细节&小技巧

  1. 有的时候如果找不到合适的”pop|ret“，可以寻找”pop|pop|…|ret”，但构建payload时要记得把相应值填上
  2. 注意sendlne时结尾自带换行符，send结尾不带换行符（这在泄露ebp的时候有影响）

漏洞利用

程序崩溃

通常来说，利用格式化字符串漏洞使得程序崩溃是最为简单的利用方式，因为我们只需要输入若干个 %s 即可

%s%s%s%s%s%s%s%s%s%s%s%s%s%s

这是因为栈上不可能每个值都对应了合法的地址，所以总是会有某个地址可以使得程序崩溃。这一利用，虽然攻击者本身似乎并不能控制程序，但是这样却可以造成程序不可用。比如说，如果远程服务有一个格式化字符串漏洞，那么我们就可以攻击其可用性，使服务崩溃，进而使得用户不能够访问。

泄露内存

利用格式化字符串漏洞，我们还可以获取我们所想要输出的内容。一般会有如下几种操作

  • 泄露栈内存
    • 获取某个变量的值
    • 获取某个变量对应地址的内存
    • 修改栈内存上地址指向数据的值：第一步，泄露出字符串在栈上的地址；第二步，将地址写进字符串；第三步，利用%$定位字符串上的这个地址，并修改其指向的值
  • 泄露任意地址内存
    • 利用 GOT 表得到 libc 函数地址，进而获取 libc，进而获取其它 libc 函数地址
    • 盲打，dump 整个程序，获取有用信息。

Linux保护机制及其漏洞和绕过方法

RELRO

RELRO（ReLocation Read-Only）是一种安全机制，通过将程序的重定位表设置为只读来防止GOT（全局偏移表）覆写攻击，分为Partial RELRO（部分只读）和Full RELRO（完全只读），前者仅保护GOT表的一部分，后者则保护所有关键数据结构

STACK CANARY

Stack Canary是一种栈保护机制，旨在防止缓冲区溢出攻击。其原理是在函数调用时，在栈帧顶部插入一个随机值（canary），并在函数返回时检查该值是否被篡改。若被篡改，则表明栈溢出发生，程序将立即停止执行。这种机制通过在函数入口处插入canary值，并在函数返回时验证其完整性来检测栈溢出。Stack Canary并非理论上绝对安全，仅能有效缓解顺序覆盖的栈溢出攻击。

def leak_canary():
    global canary
    canary = b"\x00"
    while len(canary) < 8:
        for x in range(256):
            try:
                with remote("node5.buuoj.cn", 26857) as io:
                    payload = b"A" * (0x5C) + canary + bytes([x])
                    io.sendline(payload)
                    io.recv()
                    canary += bytes([x])
                    break
            except EOFError:
                continue
            except Exception as e:
                log.error(f"Unexpected error: {e}")
                continue
    log.success(f"Canary: {canary.hex()}")

NX

NX（No-eXecute）是一种安全机制，旨在防止攻击者在堆栈中执行恶意代码。其基本原理是将数据所在内存页标识为不可执行，当程序溢出成功转入shellcode时，程序会尝试在数据页面上执行指令，此时CPU就会抛出异常，而不是去执行恶意指令。NX的实现由软件和硬件共同完成，硬件层利用CPU的NX位，软件层面需要操作系统支持NX以配置页表

ASLR和PIE

ASLR（Address Space Layout Randomization）和PIE（Position-Independent Executable）是计算机安全领域的重要概念，旨在增强程序的安全性。ASLR通过随机化程序在内存中的地址布局，防止攻击者预测内存地址，从而降低缓冲区溢出攻击的成功率。PIE则允许程序在内存中任意位置执行，不依赖于固定的内存地址，这与ASLR结合使用，进一步增强了程序的安全性。

FORTIFY_SOURCE

FORTIFY_SOURCE 是 GCC 和 Clang 编译器提供的一个编译选项，旨在增强 C 和 C++ 程序的安全性，通过在编译期间插入额外的检查来防止常见的缓冲区溢出和其他类型的内存损坏。它通过定义宏 _FORTIFY_SOURCE 来控制对一些函数的调用进行加固，确保在编译时和运行时检测缓冲区溢出和格式化字符串漏洞。FORTIFY_SOURCE 提供两个级别：-D_FORTIFY_SOURCE=1 进行较弱的检查，-D_FORTIFY_SOURCE=2 进行较强的检查，后者在运行时也会进行检查，以防止缓冲区溢出。
