git学习笔记

  1. 新建仓库
git init
git init Forder_name
git clone https://..../xxx.git
  1. 工作区域和文件状态
工作区域 英文
工作区 Working Directory
暂存区 Staging Area/Index
本地仓库 Local Repository
文件状态 英文
未跟踪 Untrack
未修改 Unmodified
已修改 Modified
已暂存 Staged 
git status
  1. 添加和提交仓库
git add File_name # 添加跟踪文件
git add .
git add *.txt

git rm --cached File_name # 取消跟踪文件

git ls-files # 查看暂存区中的文件

git commit -m Commit_infomation # 提交暂存区中的文件

git log # 查看提交记录
git log --oneline # 查看简洁的提交记录
git reflog # 查看所有版本
  1. 回退版本
git reset --soft ID
git reset --hard ID
git reset --mixed ID
  1. 查看差异
git diff                        # 工作区——暂存区
git diff HEAD                   # 工作区——版本库
git diff --cached               # 暂存区——版本库
git diff ID1 ID2 (File_name)    # 版本库——版本库

HEAD    #最新提交版本
HEAD~   #上一个版本
HEAD^   #上一个版本
HEAD~n  #上n个版本
  1. 删除文件
rm File_name; git add File_name # 工作区->暂存区
git rm File_name                # 工作区，暂存区
git rm --cached File_name       # 暂存区
git rm -r *                     # 递归删除某个目录下的所有子目录和文件

git commit
  1. 忽略文件

.gitignore文件，前提是这些文件不在暂存区中

*.log
*.o
*.class
*.env
*.zip
*.tar
*.pem
...

.gitignore文件的匹配规则：

  • 空行或者以#开头的行会被Git忽略。一般空行用于可读性的分隔,#一般用作注释
  • 星号 * 通配任意个字符
  • 问号 ? 匹配单个字符
  • 中括号 [] 表示匹配列表中的单个字符,比如:[abc] 表示a/b/c
  • 两个星号 ** 表示匹配任意的中间目录
  • 中括号可以使用短中线 - 连接，比如:[0-9]表示任意一位数字,[a-z]表示任意一位小写字母
  • 感叹号 ! 表示取反
